# スキル定義: 商品マスタ画面 実装・検証

## 1. スキルの目的
Book2 系ドキュメントに基づき、商品マスタ画面の要件を実装へ落とし込み、画面側とサーバ側を整合させて動作確認まで完了させる。

## 2. 対象範囲
- 画面: `mock/index.html`, `mock/menu01.html`, `mock/styles.css`, `mock/script.js`
- サーバ: `server/app.py`
- 仕様: `Book2_画面.md`, `Book2_実装計画書.md`

## 3. 期待アウトプット
- 仕様と実装差分が明確な実装メモ
- CRUD API と画面操作の対応表
- 最低限の動作確認結果（成功/失敗ケース）

## 4. 前提知識
- HTML/CSS/JavaScript の基本
- Flask での REST API 実装
- PostgreSQL の CRUD 操作
- 画面状態管理（ボタン活性/非活性、入力値リセット）

## 5. 実行手順
1. 仕様確認
	- Book2 ドキュメントから画面遷移、一覧表示、入力欄、ボタン制御、終了動作を抽出する。
2. 画面実装
	- 一覧 7 行固定、8 件目以降スクロール、列幅一致を調整する。
	- コード入力フォーカスアウト時に存在チェックを行い、名称/金額を反映する。
	- 状態ごとに作成・更新・削除・終了ボタンを切り替える。
3. API 実装
	- `GET /products`, `GET /products/{code}`, `POST /products`, `PUT /products/{code}`, `DELETE /products/{code}` を実装する。
	- `price` と `kingaku` のマッピングを統一する。
4. 結合確認
	- 画面から API を呼び出し、一覧再表示とエラーメッセージ表示を確認する。
5. 完了判定
	- 実装計画書の完了条件を満たしたことを確認する。

## 6. 品質基準
- 仕様外の画面遷移がない
- CRUD 成功時に一覧が再取得される
- 失敗時に利用者向けメッセージが表示される
- 初期状態と処理後状態でボタン制御が一致する
- 実装で追加・修正したコードには、何の作業をしているかが分かる簡潔なコメントを必要箇所に残す

## 7. 受け入れチェックリスト
- [ ] メニューから商品マスタ画面へ遷移できる
- [ ] 7行以内は固定表示、8件以上でデータ部のみスクロールする
- [ ] コード既存/未存在で名称・金額とボタン状態が切り替わる
- [ ] 作成・更新・削除後に一覧が正しく更新される
- [ ] 終了操作で想定どおりに画面遷移または終了する
- [ ] 追加・修正したロジックの要所に、作業意図が分かるコメントがある

## 8. 失敗パターン
- 画面項目名と DB 項目名の不一致（`price` と `kingaku`）
- ボタンの表示/非表示で制御し、レイアウトが崩れる
- API 失敗時に無言で処理が終わる
