# 商品マスタ画面 実装計画書

## 1. 目的
Book2_画面.md の記載内容を基に、以下を実現する。
- メニュー画面（index.html）から商品マスタ画面（menu01.html）へ遷移できること
- 商品マスタ画面で CRUD（作成・更新・削除）を実行できること
- 終了操作で画面を閉じる、またはメニューへ戻ること

## 2. 対象ファイル
- 画面側
  - `mock/menu01.html`
  - `mock/styles.css`
  - `mock/script.js`
- サーバ側
  - `server/app.py`
- 仕様書
  - `Book2_画面.md`

## 3. 機能要件（Book2 反映）
### 3.1 画面遷移
- `index.html` に「商品マスタ画面」メニューを配置する。
- メニュークリックで `menu01.html` へ遷移する。

### 3.2 商品一覧表示
- 初期表示時に DB テーブル `s_master` から一覧取得する。
- 表示件数は 7 行を基準とし、8 件目以降はデータ部のみ縦スクロール表示する。
- ヘッダー（コード/名称/金額）とデータ列幅は一致させる。

### 3.3 入力欄
- 入力項目: `code`, `name`, `price`（DB は `kingaku`）
- コード・金額は右寄せ表示。
- コード入力後、フォーカスアウト時に DB 存在チェックを行う。
  - 存在する: 名称/金額を自動表示
  - 存在しない: 名称/金額を初期化

### 3.4 ボタン制御
- ボタンは常時表示、活性/非活性のみ切り替える。
- 初期: 終了のみ活性
- 一覧選択時: 更新・削除・終了を活性、作成は非活性
- コード未存在時: 作成・終了を活性、更新・削除は非活性
- 作成/更新/削除 実行後: 初期状態（終了のみ活性）へ戻す

### 3.5 CRUD
- 作成: `POST /products`
- 更新: `PUT /products/{code}`
- 削除: `DELETE /products/{code}`
- 1件取得: `GET /products/{code}`
- 一覧取得: `GET /products`

### 3.6 終了
- 終了ボタン押下時の仕様を確定する。
  - 案A: 画面を閉じる（ブラウザ制約により不可の場合あり）
  - 案B: `index.html` へ戻る（推奨）

## 4. 非機能要件
- レスポンス失敗時に利用者向けメッセージを表示する。
- 文字化け防止のため UTF-8 を使用する。
- 開発環境は Flask + PostgreSQL + 静的 HTML/CSS/JS を前提とする。

## 5. 実装タスク
### 5.1 画面側
1. `menu01.html` の構造維持（一覧、入力、ボタン）
2. `styles.css` で以下を調整
   - 一覧の 7 行表示固定
   - データ部のみスクロール
   - ヘッダーとデータ列幅一致
   - 入力欄の code/price 右寄せ
3. `script.js` に状態管理を実装
   - 初期/既存コード/新規コードのボタン状態
   - フォーカスアウト時のコード存在チェック
   - CRUD 後の状態リセット

### 5.2 サーバ側
1. `app.py` に API を実装
   - `GET /products`
   - `GET /products/<code>`
   - `POST /products`
   - `PUT /products/<code>`
   - `DELETE /products/<code>`
2. DB マッピング
   - 画面 `price` <-> DB `kingaku`
3. CORS 設定
   - 画面から API 呼び出し可能にする

### 5.3 メニュー画面
1. `index.html` 新規作成（または更新）
2. 商品マスタ画面への遷移リンクを追加

## 6. テスト計画
### 6.1 単体観点
- 一覧取得が成功/失敗したときの表示
- コード存在/未存在での名称・金額とボタン状態
- 作成/更新/削除の成功/失敗

### 6.2 画面観点
- 7件以下でレイアウト崩れがない
- 8件以上でスクロールバー表示、列ズレなし
- 入力欄とボタンの位置が固定される

### 6.3 DB観点
- 作成後に `s_master` にINSERTされる
- 更新後に対象コードのみ更新される
- 削除後に対象コードのみ削除される

## 7. リスク・留意点
- ブラウザを `file://` で開く場合、環境により制約が出る可能性がある。
- 終了ボタンで「タブを閉じる」はブラウザ仕様上、動作制約がある。
- スクロールバー幅は OS/ブラウザ差があるため、必要に応じて微調整する。

- 商品マスタ画面の CRUD とボタン活性制御が要件どおりに動作する。
- 7行表示・8件目スクロール時のレイアウトが崩れない。
- テスト観点の主要シナリオがすべて合格する。
